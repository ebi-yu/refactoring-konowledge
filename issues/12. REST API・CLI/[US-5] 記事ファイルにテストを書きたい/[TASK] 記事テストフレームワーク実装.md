## 概要

記事のMarkdownに対してアサーションを書けるテストフレームワークの実装。

## 親 STORY

- STORY: (US-5) 記事ファイルにテストを書きたい

## 作業内容

- [ ] テストフレームワークのAPI設計（`describe` / `it` / `expect`形式）
- [ ] Markdownパーサーとアサーション関数の実装（見出しチェック・リンク切れ検出・必須セクション確認等）
- [ ] テストファイル（`*.test.md.js`等）の読み込みと実行ランナーの実装
- [ ] テスト結果レポートのフォーマット実装（PASS/FAIL出力）

## 技術メモ

- アサーション例: `expect(article).toHaveSection('## まとめ')`, `expect(article).toHaveNoDeadLinks()`
- Markdownのパースはremark/unifiedエコシステムを活用する
- リンク切れ検出はHTTPリクエストで実際に確認するオプションも提供
