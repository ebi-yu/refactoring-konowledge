## ユーザーストーリー

| 項目                   | 内容                                                                                                         |
| ---------------------- | ------------------------------------------------------------------------------------------------------------ |
| **Who**（誰が）        | 投稿者                                                                                                       |
| **What**（何をしたい） | 記事ファイルに対してテストスクリプトを書いて実行したい                                                       |
| **Why**（なぜ・価値）  | 構成・リンク切れ・必須セクションの有無など投稿基準をコードで定義して、公開前に自動検証できるようにしたいから |

**ストーリー文（上記を文章化）:**

> **投稿者** として、**記事ファイルに対してテストスクリプトを書いて実行したい**。なぜなら **構成・リンク切れ・必須セクションの有無など投稿基準をコードで定義して、公開前に自動検証できるようにしたい** からだ。

## 背景・補足

Ph2スコープ。`knowledge test` CLIコマンドで記事MarkdownファイルにアサーションベースのテストスクリプトをCIに組み込み、品質ゲートとして活用できるようにする。
