## 概要

認証ミドルウェアにゲストアクセス許可ルールを設定する実装（公開記事は認証不要）。

## 親 STORY

- STORY: [US-3] ログインせずに公開記事を閲覧したい

## 作業内容

- [ ] 認証ミドルウェアに「公開記事ルートはスキップ」する条件分岐を追加する
- [ ] ルート定義に public / protected の区別を明示する（例: ルートメタまたはデコレーター）
- [ ] 未認証ユーザーが protected ルートにアクセスした場合に `/login` へリダイレクトするロジックを実装する
- [ ] API エンドポイントでも公開記事取得は認証不要、それ以外は 401 を返すよう設定する

## 技術メモ

- ルートごとの公開・非公開設定はルートメタフィールド（例: `meta: { requiresAuth: true }`）で管理する
- SPA のクライアントサイドルーティングと SSR/API の両方でアクセス制御を一貫して適用する
