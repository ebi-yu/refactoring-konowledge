## 概要

メールアドレスとパスワードを受け取り、認証を行うログインAPIエンドポイントを実装する。
認証成功時にはセッションまたはJWTトークンを発行し、失敗時には適切なエラーレスポンスを返す。

## 技術メモ

- パスワードハッシュには bcrypt を使用する
- セッション管理は既存のセッション管理実装（[TASK] セッション管理実装）と統一する
- エンドポイント: `POST /api/auth/login`
- リクエスト形式:
  ```json
  {
    "email": "user@example.com",
    "password": "secret"
  }
  ```
- レスポンス（成功時 200 OK）:
  ```json
  {
    "token": "<jwt_or_session_token>",
    "user": { "id": "...", "email": "...", "name": "..." }
  }
  ```
- レスポンス（失敗時 401 Unauthorized）:
  ```json
  {
    "error": "メールアドレスまたはパスワードが正しくありません"
  }
  ```
