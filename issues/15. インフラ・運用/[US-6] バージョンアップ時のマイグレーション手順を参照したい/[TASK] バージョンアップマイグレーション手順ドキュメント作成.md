## 概要

バージョンアップ時の DB マイグレーション・設定変更・Docker イメージ更新の手順をまとめた `UPGRADE.md` ドキュメントの作成。

## 親 STORY

## 作業内容

- [ ] `UPGRADE.md`（またはdocs/upgrade-guide.md）を作成する
- [ ] バージョン間の変更点（破壊的変更・新設定項目）を記載するセクションを設ける
- [ ] DB マイグレーション実行手順（`docker compose run app npm run migrate` 等）を明記する
- [ ] .env への新規追加設定項目の一覧とデフォルト値を記載する
- [ ] ロールバック手順（DB リストア + 旧イメージ起動）を記載する
- [ ] 最初のリリース（v1.0）向けの手順を作成してドキュメントを検証する

## 技術メモ

- マイグレーションは DB マイグレーションツール（例: Knex.js、Prisma Migrate、Flyway）を使用し、バージョン管理する
- セマンティックバージョニングに従い、破壊的変更はメジャーバージョンアップとする
- ロールバック手順には EPIC 15 US-5 のバックアップスクリプトを活用する
