# [EPIC] 技術的TODO

## 概要

各 EPIC の実装に先立って対応が必要な技術的課題や、将来対応を保留した技術的負債を管理する。
機能開発 EPIC に属さないランタイム最適化・ライブラリアップグレード・アーキテクチャ改善などの横断的なタスクを集約する。

## EPIC ストーリー

**開発者** として、**技術的負債や将来の最適化課題を一元管理し、適切なタイミングで対処** したい。なぜなら **機能開発 EPIC に散らばった技術課題を見落とさず、システム全体の品質を継続的に向上させたい** からだ。

## スコープ（STORY 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

| 優先度  | #   | ストーリー（一文）                                                                                                                         | Issue |
| ------- | --- | ------------------------------------------------------------------------------------------------------------------------------------------ | ----- |
| `Could` | 1   | 開発者として、Nuxt の Vapor Mode に対応したい。なぜならVue の仮想 DOM オーバーヘッドを削減してランタイムパフォーマンスを向上させたいから。 | -     |

## スコープ外

- **機能要件に紐づく技術タスク**: 各機能 EPIC の STORY として管理する
- **インフラ・運用関連の技術課題**: 15. インフラ・運用 EPIC で管理する

## 受け入れ基準（EPIC 完了の定義）

- [ ] 配下のすべての `Must` STORY が Done になっている
- [ ] 各 STORY の完了定義（DoD）をそれぞれ満たしている
