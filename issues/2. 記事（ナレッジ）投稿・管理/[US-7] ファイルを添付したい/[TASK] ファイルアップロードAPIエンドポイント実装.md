## 概要

POST /api/attachments エンドポイントの実装（マルチパートフォームデータ受け取りとストレージ保存）。

## 親 STORY

## 作業内容

- [ ] `POST /api/attachments` エンドポイントを実装する（`multipart/form-data` 受け取り）
- [ ] アップロード可能なファイルタイプ（画像・PDF 等）とサイズ上限（例: 10MB）を設定する
- [ ] ファイルをストレージに保存し、公開 URL を DB の attachments テーブルに記録する
- [ ] レスポンスでファイルの URL と Markdown 挿入用テキストを返す
- [ ] ウイルススキャンまたはファイルタイプ検証を実装する

## 技術メモ

- ファイル名は UUID で一意に生成してストレージに保存する（元ファイル名は DB のみに保存）
- アップロードサイズ制限はリバースプロキシ（nginx）と API の両方で設定する
