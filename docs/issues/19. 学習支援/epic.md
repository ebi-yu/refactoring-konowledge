# [EPIC] 学習支援（Claude Code Plugin）

## 概要

記事をローカルで Markdown 管理する投稿者向けの Claude Code プラグイン。
CLI 導入時に一緒にインストールされ、「想定読者として読む AI エージェント」が記事にフィードバックを返す。
「AIに書かせる」ではなく、**書いた記事が想定読者に届くかを確認するための学習支援**として機能する。
**全機能が Ph2 スコープ。**

## EPIC ストーリー

**投稿者** として、**書き上げた記事を想定読者の視点で AI に読んでもらい、フィードバックをもらいたい**。なぜなら **書き手の視点だけでは気づけない「読者が詰まる箇所」や「前提知識のズレ」を公開前に発見して、伝わる記事に仕上げたいから**。

## スコープ（STORY 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば
> **⚠️ このEPICは Ph2 スコープのため Ph1 リリースには含まれない**

| 優先度  | #   | ストーリー（一文）                                                                                                                                                                               | Issue |
| ------- | --- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----- |
| `Must`  | 1   | 投稿者として、想定読者のペルソナ（初心者・熟練者など）を指定して記事のフィードバックをもらいたい。なぜなら書き手では気づけない「読者が詰まる箇所」や「前提知識のズレ」を公開前に発見したいから。 | -     |
| `Could` | 2   | 投稿者として、複数のペルソナから並行してレビューをもらいたい。なぜなら記事が多様な読者に届くかを一度に確認したいから。（Ph2）                                                                    | -     |

## スコープ外

- **AI による記事自動生成・修正**: スコープアウト。フィードバックの採否は投稿者が判断する
- **Web UI 上でのレビュー機能**: スコープアウト。ローカル執筆フローに特化

## 受け入れ基準（EPIC 完了の定義）

- [ ] 配下のすべての `Must` STORY が Done になっている
- [ ] `knowledge review --audience=<ペルソナ> article.md` でフィードバックが返るシナリオが通過している
- [ ] Claude Code プラグインとして CLI 導入手順に含まれている
