## ユーザーストーリー

| 項目                   | 内容                                                   |
| ---------------------- | ------------------------------------------------------ |
| **Who**（誰が）        | 投稿者                                                 |
| **What**（何をしたい） | タイトルとMarkdown本文を入力して新しい記事を作成したい |
| **Why**（なぜ・価値）  | 自分の知識やノウハウをチームメンバーに届けたいから     |

**ストーリー文（上記を文章化）:**

> **投稿者** として、**タイトルとMarkdown本文を入力して新しい記事を作成したい**。なぜなら **自分の知識やノウハウをチームメンバーに届けたいから** だからだ。

## 背景・補足

記事作成はシステムのコア機能。ログイン済みユーザーがタイトルと Markdown 本文を入力し投稿できる。投稿後は記事詳細ページにリダイレクトされ、Markdown がレンダリングされた状態で確認できる。

## 受け入れ基準

- [ ] **Given** ログイン済みユーザー  
       **When** 記事作成ページでタイトルと本文を入力して投稿ボタンを押す  
       **Then** 記事がDBに保存され、記事詳細ページにリダイレクトされる

- [ ] **Given** 投稿した記事  
       **When** 記事詳細ページにアクセスする  
       **Then** タイトルとMarkdownがレンダリングされた本文が表示される
