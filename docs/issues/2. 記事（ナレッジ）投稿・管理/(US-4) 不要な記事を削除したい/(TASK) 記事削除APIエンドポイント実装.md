## 概要

DELETE /api/articles/:id エンドポイントの実装（自分の記事のみ削除可能な権限チェック含む）。

## 親 Story

- Story: (US-4) 不要な記事を削除したい

## 作業内容

- [ ] `DELETE /api/articles/:id` エンドポイントを実装する
- [ ] `article.author_id !== session.userId` の場合は 403 を返す
- [ ] 存在しない記事 ID の場合は 404 を返す
- [ ] 記事に紐づく添付ファイル・コメント・リアクション・ストックも CASCADE で削除する
- [ ] 削除成功時は 204 No Content を返す

## 技術メモ

- 物理削除ではなく `deleted_at` を設定するソフトデリートを検討する（削除取り消し機能の余地）
- 管理者は他者の記事も削除できるよう権限チェックを拡張可能にしておく
