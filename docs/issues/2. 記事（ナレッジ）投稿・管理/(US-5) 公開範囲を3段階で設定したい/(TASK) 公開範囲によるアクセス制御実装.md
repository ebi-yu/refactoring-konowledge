## 概要

記事取得 API およびページルーティングで公開範囲に応じたアクセス制御を実装。

## 親 Story

- Story: (US-5) 公開範囲を3段階で設定したい

## 作業内容

- [ ] `GET /api/articles/:id` で visibility チェックを実装する
  - `public`：認証不要で 200 を返す
  - `login_only`：未認証の場合 401 を返す
  - `private`：記事の author_id が session.userId と一致しない場合 403 を返す
- [ ] `GET /api/articles`（一覧）で、ログイン状態に応じて表示できる記事のみフィルタリングする
- [ ] フロントエンドのルートガードで visibility に応じたリダイレクトを実装する

## 技術メモ

- 管理者は全記事にアクセスできるよう権限ロジックを整理する
- パフォーマンス最適化のため、visibility フィールドに DB インデックスを付与する
