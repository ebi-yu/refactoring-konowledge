# [EPIC] スレッド（スクラップ）

## 概要

短いメモ・断片的な知識をスレッド形式で積み重ねる軽量な投稿機能。
記事（Markdown）より手軽で、作業ログ・調査メモ・Q&A の記録など「まとまる前の知識」をためる場として使う。
Ph1 からコア機能を提供する。

## EPIC ストーリー

**投稿者** として、**正式な記事にする前の断片的なメモや調査ログをスレッド形式で気軽に書き溜めたい**。なぜなら **完成度を求めると投稿のハードルが上がって知識が埋もれてしまうから**。

## スコープ（STORY 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

| 優先度  | #   | ストーリー（一文）                                                                                                                                 | Issue |
| ------- | --- | -------------------------------------------------------------------------------------------------------------------------------------------------- | ----- |
| `Must`  | 1   | 投稿者として、トピックタイトルを付けてスレッドを作成したい。なぜなら調査や作業ログをテーマごとにまとめて書き溜めたいから。                         | -     |
| `Must`  | 2   | 投稿者として、スレッドに短いメモやコードスニペットを時系列で追記したい。なぜなら思いついたことや調べたことをその都度記録して後から参照したいから。 | -     |
| `Must`  | 3   | 投稿者として、スレッドを公開 / 非公開に設定したい。なぜなら作業中のメモは自分だけ見えるようにしておき、まとまったら公開したいから。                | -     |
| `Could` | 4   | 投稿者として、スレッドの内容をまとめて正式な記事に変換したい。なぜなら書き溜めたメモを整理して読みやすい記事として発信したいから。（Ph2）          | -     |

## スコープ外

- **記事への昇格**（13-26）: Ph2

## 受け入れ基準（EPIC 完了の定義）

- [ ] 配下のすべての `Must` STORY が Done になっている
- [ ] スレッド作成・投稿追記・公開設定の E2E シナリオが通過している
- [ ] ドキュメント（ADR / API仕様）が更新されている
