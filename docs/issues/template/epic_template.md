# Epic テンプレート

> 機能グループ単位で起票する Issue のテンプレート。
> `01_feature_scope.md` の章（`### X. xxxxxx`）1 つが 1 Epic に対応する。

---

## タイトル形式

```
[Epic] <機能グループ名>
例: [Epic] 認証・アカウント管理
```

## ラベル

`type:epic`

---

## Issue 本文

```markdown
## 概要

<!-- この Epic が扱う機能グループの目的・価値を 2〜3 文で説明する -->

## Epic ストーリー

<!-- ⚠️ Who / What / Why はすべて必須。空欄のまま起票しないこと。 -->

**〈Who〉** として、**〈What〉** がしたい。なぜなら **〈Why〉** だからだ。

## スコープ（Story 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

| 優先度   | #   | ストーリー（一文）                                       | Issue   |
| -------- | --- | -------------------------------------------------------- | ------- |
| `Must`   | 1   | 〈Who〉として、〈What〉がしたい。なぜなら〈Why〉だから。 | #<番号> |
| `Must`   | 2   | 〈Who〉として、〈What〉がしたい。なぜなら〈Why〉だから。 | #<番号> |
| `Should` | 3   | 〈Who〉として、〈What〉がしたい。なぜなら〈Why〉だから。 | #<番号> |
| `Could`  | 4   | 〈Who〉として、〈What〉がしたい。なぜなら〈Why〉だから。 | #<番号> |

## スコープ外

<!-- 意図的に含めないもの・将来の Epic で扱うものを明記する -->

- ...

## 受け入れ基準（Epic 完了の定義）

- [ ] 配下のすべての `Must` Story が Done になっている
- [ ] E2E シナリオが通過している
- [ ] ドキュメント（ADR / API仕様）が更新されている

## 参照

- [[01_feature_scope]] - 機能スコープ一覧（セクション番号: X）
- [[ADR-XXX]] - 関連する技術決定
```
