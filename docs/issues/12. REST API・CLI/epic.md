# [EPIC] REST API / CLI（外部連携）

## 概要

外部システムやコマンドラインから記事・ユーザー・ファイルを操作できる REST API と CLI ツールの機能群。
**全機能が Ph2 スコープ。** Ph1 でコア機能を固めた後、外部連携の窓口を開く。

## EPIC ストーリー

**開発者** として、**コマンドラインや CI/CD パイプラインから記事を投稿・更新したい**。なぜなら **毎回 Web UI を開かずに手元のエディタや自動化スクリプトから直接ナレッジを更新できると、投稿の手間が減って継続しやすくなるから**。

## スコープ（STORY 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば
> **⚠️ このEPICは Ph2 スコープのため Ph1 リリースには含まれない**

| 優先度   | #   | ストーリー（一文）                                                                                                                                                                           | Issue |
| -------- | --- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----- |
| `Must`   | 1   | 開発者として、REST API で記事の作成・取得・更新・削除をしたい。なぜなら外部ツールや自動化スクリプトとシステムを連携させたいから。                                                            | -     |
| `Must`   | 2   | 開発者として、REST API でファイルのアップロード・取得をしたい。なぜなら記事に必要な画像や資料をプログラムから添付したいから。                                                                | -     |
| `Should` | 3   | 開発者として、REST API でユーザーやグループの情報を取得したい。なぜなら外部システムとメンバー情報を同期したいから。                                                                          | -     |
| `Could`  | 4   | 開発者として、CLI ツールで記事の作成・投稿・編集をターミナルから操作したい。なぜなら普段使っているエディタで書いた記事をそのままコマンド一発で投稿したいから。                               | -     |
| `Could`  | 5   | 投稿者として、記事ファイルに対してテストスクリプトを書いて実行したい。なぜなら構成・リンク切れ・必須セクションの有無など投稿基準をコードで定義して、公開前に自動検証できるようにしたいから。 | -     |

## スコープ外

- Ph1 では REST API は提供しない（Nuxt サーバーの内部 API のみ）
- 認証には APIトークン（1-8）が必要なため、1-8 と同時に実装する

## 受け入れ基準（EPIC 完了の定義）

- [ ] 配下のすべての `Must` STORY が Done になっている
- [ ] REST API（記事 CRUD・ファイル操作）の E2E シナリオが通過している
- [ ] OpenAPI 仕様書が生成・更新されている
