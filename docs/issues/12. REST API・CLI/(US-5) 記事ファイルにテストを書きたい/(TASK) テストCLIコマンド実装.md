## 概要

## 親 Story

- Story: (US-5) 記事ファイルにテストを書きたい

## 作業内容

- [ ] `knowledge test <file>` コマンドのCLIエントリーポイント実装
- [ ] テストファイルの自動検出（`<article>.test.js` 規約）
- [ ] テスト実行結果の色付きターミナル出力実装
- [ ] 終了コード（成功: 0 / 失敗: 1）の設定（CI連携用）

## 技術メモ

- テストファイルが存在しない場合は警告を出して終了コード0で終わる
- `--watch` オプションでファイル変更時に自動再実行できるようにする
- CI環境での利用を想定してJUnit XML形式のレポート出力オプションも追加する
