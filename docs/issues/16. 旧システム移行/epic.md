# [EPIC] 旧システム移行

## 概要

旧システム（Java版 knowledge）からの記事・添付ファイルの移行ツール・手順を提供する。
移行は一括バッチ実行を基本とする。

## EPIC ストーリー

**システム管理者** として、**旧システムの記事と添付ファイルを新システムに移行して、過去の知識をそのまま使い続けられるようにしたい**。なぜなら **蓄積されてきた記事が失われると移行する意味がなくなってしまうから**。

## スコープ（STORY 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

| 優先度 | #   | ストーリー（一文）                                                                                                                                  | Issue |
| ------ | --- | --------------------------------------------------------------------------------------------------------------------------------------------------- | ----- |
| `Must` | 1   | 管理者として、旧システムの記事データ・タグを移行したい。なぜなら過去に蓄積した知識を新システムでもそのまま活用したいから。                          | -     |
| `Must` | 2   | 管理者として、旧システムの添付ファイル（DB の BYTEA）をクラウドストレージに移行したい。なぜなら画像や資料が失われると記事の内容が不完全になるから。 | -     |

## スコープ外

- **ユーザーデータ移行**（13-17）: 範囲外
- **コメント・リアクション移行**（13-20）: 範囲外
- **移行検証・レポート**（13-22）: 範囲外
- **グループ・権限移行**（13-21）: グループ機能除外に伴いスコープアウト

## 受け入れ基準（EPIC 完了の定義）

- [ ] 配下のすべての `Must` STORY が Done になっている
- [ ] 記事・添付ファイル移行の E2E シナリオが通過している
- [ ] ドキュメント（ADR / API仕様）が更新されている
