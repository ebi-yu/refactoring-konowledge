# [EPIC] 検索・発見

## 概要

投稿者・閲覧者が目的の記事にたどり着くための検索と発見機能群。
全文検索・タグ絞り込み・新着フィード・トレンド表示を Ph1 で提供し、関連記事サジェストは Ph2 で追加する。
トレンドは累計でなく「今伸びている」記事を時間減衰加重スコアで可視化する。

## EPIC ストーリー

**閲覧者** として、**知りたいことをすぐに検索で見つけたり、見逃していた新着・話題の記事を自然に発見したい**。なぜなら **膨大な記事の中から必要な情報を探し回るのは時間の無駄だし、面白い記事に気づけないのももったいないから**。

## スコープ（STORY 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

| 優先度  | #   | ストーリー（一文）                                                                                                                                              | Issue |
| ------- | --- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----- |
| `Must`  | 1   | 閲覧者として、キーワードを入力して記事のタイトル・本文・タグを横断検索したい。なぜなら必要な情報がどこに書いてあるか分からないときでも素早く見つけたいから。    | -     |
| `Must`  | 2   | 閲覧者として、トップページで公開日時順の新着記事一覧を見たい。なぜなら最近チームに共有された情報をチェックして乗り遅れないようにしたいから。                    | -     |
| `Must`  | 3   | 閲覧者として、直近 24 時間・7 日間で急に読まれている「勢い記事」を確認したい。なぜなら今チームの中で注目されているトピックを手軽に把握したいから。              | -     |
| `Could` | 4   | 閲覧者として、キーワードの意味に近い記事を自然言語で検索したい。なぜなら正確な言葉を知らなくても意図した記事にたどり着きたいから。（Ph2・Embedding + pgvector） | -     |

## スコープ外

- **タグ絞り込み**（4-2）: 除外
- **関連記事表示**（4-5）: 除外
- **閲覧履歴**（4-4）: 個人の閲覧履歴管理はプライバシーコストに対して価値が低いため除外
- **検索結果ハイライト**（4-3）: 実装コストが高いため Ph1 では除外。必要になれば将来追加

## 受け入れ基準（EPIC 完了の定義）

- [ ] 配下のすべての `Must` STORY が Done になっている
- [ ] 全文検索・タグ絞り込み・ハイライト・新着フィード・トレンドの E2E シナリオが通過している
- [ ] ドキュメント（ADR / API仕様）が更新されている
