# [EPIC] Book（体系的知識）

## 概要

複数の記事を章立てで構造化してまとめる「本（Book）」機能。
チュートリアル・ハンドブック・技術解説シリーズなど、順番に読ませるコンテンツに対応する。
**全機能が Ph2 スコープ。**

## EPIC ストーリー

**投稿者** として、**複数の記事を章立てにまとめた「本」を作って公開したい**。なぜなら **連載記事やチュートリアルをバラバラに投稿するより、体系的にまとめた方が読者が順番に理解しやすいから**。

## スコープ（STORY 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば
> **⚠️ このEPICは Ph2 スコープのため Ph1 リリースには含まれない**

| 優先度  | #   | ストーリー（一文）                                                                                                                                 | Issue |
| ------- | --- | -------------------------------------------------------------------------------------------------------------------------------------------------- | ----- |
| `Must`  | 1   | 投稿者として、タイトル・説明・カバー画像を持つ「本」を作成・編集・削除したい。なぜなら連載記事やチュートリアルを一つにまとめて読者に届けたいから。 | -     |
| `Must`  | 2   | 投稿者として、本にチャプターを追加・並び替え・削除したい。なぜなら章立てで内容を構成して読者が順番に理解できるようにしたいから。                   | -     |
| `Must`  | 3   | 投稿者として、本単位で公開 / 非公開 / 限定公開を設定したい。なぜなら完成した本だけを読者に届けて、書きかけのものは隠しておきたいから。             | -     |
| `Could` | 4   | 閲覧者として、本の目次をサイドバーで見ながら前後チャプターへ移動したい。なぜなら長い本を章立てで快適に読み進めたいから。（Ph2）                    | -     |

## スコープ外

- **目次・ナビゲーション**（13-15）: Ph2。Ph1 は本+チャプター一覧で十分

## 受け入れ基準（EPIC 完了の定義）

- [ ] 配下のすべての `Must` STORY が Done になっている
- [ ] Book作成・チャプター管理・公開設定の E2E シナリオが通過している
- [ ] ドキュメント（ADR / API仕様）が更新されている
