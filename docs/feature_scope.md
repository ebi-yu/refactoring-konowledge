### 1. 認証・アカウント管理

> 認証の一次手段は**外部プロバイダー（GitHub / GitLab / Keycloak）による OAuth / OIDC**。
> ユーザーの自己登録は行わず、外部プロバイダーのアカウントでのみログイン可能。
> 初回管理者アカウントはセットアップスクリプトで作成する。
> LDAP / AD はアプリ直接実装せず、Keycloak経由で吸収する。

| # | 機能 | 概要 | 旧 | スコープ | 備考 |
|---|------|------|----|---------|------|
| 1-1 | 外部プロバイダーログイン | GitHub / GitLab / Keycloak いずれかの OAuth / OIDC でログイン | 旧 | `[x]` | better-auth が OAuth/OIDCクライアントとして動作。env変数で有効プロバイダーを切り替え |
| 1-2 | 仮登録メール認証 | 登録時にメールでアクティベーション | 旧 | `[-]` | 自己登録廃止のため不要 |
| 1-3 | パスワードリセット | メール送信でパスワード再設定 | 旧 | `[-]` | 外部プロバイダーが認証を担うため不要 |
| 1-4 | プロフィール自動同期 | ログイン時に外部プロバイダーの表示名・アバターを自動取得 | 旧 | `[x]` | 手動編集なし。プロバイダー側で変更すれば次回ログイン時に反映 |
| 1-5 | LDAP / Active Directory 認証 | 企業内AD経由でのSSO | 旧 | `[-]` | アプリ直接実装なし。Keycloak連携経由で吸収 |
| 1-6 | ユーザー別設定 | 言語・テーマ等の個人設定 | 旧 | `[-]` | 言語はブラウザ設定、ダークモードはOSの設定に委譲 |
| 1-7 | ログイン履歴 | 過去のログイン日時・IPの閲覧 | 旧 | `[-]` | 外部プロバイダー側で管理 |
| 1-8 | APIトークン発行 | REST API用の個人アクセストークン管理 | 旧 | `[x2]` | REST API（12-1〜）がPh2のためPh2に連動 |
| 1-9 | 初回セットアップ（管理者アカウント作成） | `setup` スクリプトで最初の管理者ユーザーを作成 | 新 | `[x]` | Docker Compose 起動後に一度だけ実行するシードスクリプト |
| 1-10 | ゲストアクセス（未ログイン閲覧） | ログインなしで公開記事を閲覧できる | 旧 | `[x]` | 公開設定が「公開」の記事のみ対象。旧システムにも存在 |
