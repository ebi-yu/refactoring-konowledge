# 初期環境構築 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Nuxt 4 フルスタックアプリケーションの初期環境をレイヤード方式でリポジトリルートにセットアップする。

**Architecture:** Modular Monolith（Nuxt 4 フルスタック）。Presentation → Application → Domain ← Infrastructure のレイヤー構成。`server/` がバックエンド、`app/` がフロントエンド、`shared/` が型・スキーマ共有層。

**Tech Stack:** Nuxt 4, Bun, TypeScript (tsgo), Nitro v2, PostgreSQL 17, Meilisearch, Prisma, Zod, Panda CSS, better-auth, Pinia, Oxlint, Oxfmt, Husky, Docker Compose, Mise

---

## Task 2: Docker Compose セットアップ

**Files:**
- Create: `docker-compose.yml`
- Create: `.env.example`
- Create: `.env`
- Modify: `.gitignore`（`.env` を追加）

- [ ] **Step 1: `docker-compose.yml` を作成する**

```yaml
services:
  postgres:
    image: postgres:17
    environment:
      POSTGRES_DB: knowledge
      POSTGRES_USER: knowledge
      POSTGRES_PASSWORD: knowledge
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  meilisearch:
    image: getmeili/meilisearch:latest
    ports:
      - "7700:7700"
    environment:
      MEILI_MASTER_KEY: masterkey
    volumes:
      - meilisearch_data:/meili_data

volumes:
  postgres_data:
  meilisearch_data:
```

- [ ] **Step 2: `.env.example` を作成する**

```
DATABASE_URL=postgresql://knowledge:knowledge@localhost:5432/knowledge
MEILI_HOST=http://localhost:7700
MEILI_MASTER_KEY=masterkey
NUXT_PUBLIC_APP_URL=http://localhost:3000
```

- [ ] **Step 3: `.env` を作成する（gitignore 対象）**

```bash
cp .env.example .env
```

- [ ] **Step 4: サービスを起動して動作確認する**

Expected: `postgres` と `meilisearch` が `running` 状態で表示される
