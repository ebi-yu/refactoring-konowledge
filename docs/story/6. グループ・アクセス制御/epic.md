# [Epic] グループ・アクセス制御

## 概要

記事を特定グループのみに公開する機能と、ロールベースアクセス制御（RBAC）を提供する機能群。
**Keycloak 連携（Ph2）が前提のため、このEpic 全体が Ph2 スコープとなる。**
Ph1（組み込みモード）ではグループ機能は無効。グループの作成・メンバー管理は Keycloak 側で行い、アプリは JWT クレームを読むだけ。

## Epic ストーリー

**投稿者** として、**社内の特定グループメンバーだけが閲覧できる記事を投稿したい**。なぜなら **全社員には見せたくないが特定チームには共有したい機密度の高い情報があるから**。

## スコープ（Story 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば
> **⚠️ このEpicは Ph2（Keycloak連携必須）のため Ph1 リリースには含まれない**

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Must` | 1 | 投稿者として、記事の公開範囲を「特定グループのみ」に設定したい。なぜなら部署やプロジェクトメンバーだけに情報を届けたいから。 | - |
| `Must` | 2 | 管理者として、ユーザーに管理者・一般などのロールを割り当てて操作権限を制御したい。なぜなら権限のないユーザーが誤って重要な設定を変更するのを防ぎたいから。 | - |

## スコープ外

- **グループ作成・管理 UI**（6-1）: アプリ内 UI なし。Keycloak 管理画面で実施
- Ph1 では本 Epic の全機能が無効

- [ ] 配下のすべての `Must` Story が Done になっている
- [ ] Keycloak 連携環境でグループ限定公開・RBAC の E2E シナリオが通過している
- [ ] ドキュメント（ADR / API仕様）が更新されている
