# [Epic] コンテンツ表示・レンダリング

## 概要

Markdown で書かれた記事を読みやすく表示するためのレンダリング機能群。
コードブロックのシンタックスハイライト・Mermaid ダイアグラム・添付ファイルのプレビュー・URL カード表示・iframe 埋め込みを Ph1 で提供する。
Office ドキュメント（pptx/xlsx）プレビューは変換処理が必要なため Ph2 とする。数式（KaTeX）・絵文字サポートは除外。

## Epic ストーリー

**投稿者** として、**自分が書いた Markdown がコードや図を含めて正しく綺麗に表示されるのを確認したい**。なぜなら **読み手に内容が伝わるかどうかは見た目の品質に大きく左右されるから**。

## スコープ（Story 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Must` | 1 | 閲覧者として、見出し・リスト・リンク・コードブロックなどの Markdown 記法が正しく表示された記事を読みたい。なぜなら生のテキストではなく読みやすい形式で内容を理解したいから。 | - |
| `Must` | 2 | 閲覧者として、コードブロックがプログラミング言語ごとに色付けされた状態で表示されるのを見たい。なぜなら色がないとコードの構造が掴みにくく、読むのに時間がかかるから。 | - |
| `Must` | 3 | 投稿者として、Mermaid 記法でフローチャートや ER 図を記事内に描きたい。なぜなら図があると構成や関係を言葉だけより格段に伝えやすいから。 | - |
| `Must` | 4 | 閲覧者として、記事に添付された画像や PDF をページを離れずにプレビューしたい。なぜなら別タブを開かずにその場で内容を確認したいから。 | - |
| `Must` | 5 | 閲覧者として、記事内の URL が OGP カードとして表示されているのを見たい。なぜなら生 URL だけより内容が一目でわかってリンクを踏む前に何の記事かわかるから。 | - |
| `Must` | 6 | 投稿者として、YouTube やスライド等の外部コンテンツを Markdown 内に iframe で埋め込みたい。なぜなら動画や資料を記事から離れずにその場で見せたいから。 | - |
| `Could` | 7 | 閲覧者として、添付された pptx・xlsx をブラウザ内でプレビューしたい。なぜならダウンロードしなくてもスライドや表の内容をその場で確認したいから。（Ph2） | - |

## スコープ外

- **数式表示（KaTeX）**（3-3）: 利用者が限定的なため除外。必要になれば将来追加
- **絵文字サポート（`:smile:` 記法）**（3-5）: 実装コスト対効果が低いため除外
- **Officeドキュメントプレビュー**（3-9）: Ph2。変換処理（LibreOffice等）が必要

## 受け入れ基準（Epic 完了の定義）

- [ ] 配下のすべての `Must` Story が Done になっている
- [ ] Markdown・シンタックスハイライト・Mermaid・添付プレビュー・URLカード・iframe埋め込みの E2E シナリオが通過している
- [ ] ドキュメント（ADR / API仕様）が更新されている
