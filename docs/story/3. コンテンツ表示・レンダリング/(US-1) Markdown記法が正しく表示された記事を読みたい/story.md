## ユーザーストーリー

| 項目 | 内容 |
|------|------|
| **Who**（誰が） | 閲覧者 |
| **What**（何をしたい） | 見出し・リスト・リンク・コードブロックなどのMarkdown記法が正しく表示された記事を読みたい |
| **Why**（なぜ・価値） | 生のテキストではなく読みやすい形式で内容を理解したいから |

**ストーリー文（上記を文章化）:**

> **閲覧者** として、**見出し・リスト・リンク・コードブロックなどのMarkdown記法が正しく表示された記事を読みたい** がしたい。なぜなら **生のテキストではなく読みやすい形式で内容を理解したいから** だからだ。

## 背景・補足

記事はMarkdownで書かれているため、閲覧ページでは適切にレンダリングされた HTML として表示する必要がある。また、XSS 対策としてサニタイズも必須。

## 受け入れ基準

- [ ] **Given** Markdown本文のある記事  
      **When** 詳細ページを開く  
      **Then** 見出し・リスト・リンク・コードブロックがHTMLとして正しくレンダリングされる

- [ ] **Given** 悪意あるHTMLを含む記事  
      **When** 詳細ページを開く  
      **Then** XSSが発生しない（サニタイズ済み）
