## 概要

DB マイグレーションファイルのバージョン管理と自動実行の仕組みの実装。アップグレード時に未適用のマイグレーションが自動で実行される環境を整える。

## 親 Story

- Story: (US-6) バージョンアップ時のマイグレーション手順を参照したい

## 作業内容

- [ ] DB マイグレーションツール（Knex.js / Prisma Migrate 等）を選定し導入する
- [ ] マイグレーションファイルをバージョン番号付きで管理するディレクトリ構成を整備する（例: `migrations/YYYYMMDDHHMMSS_description.sql`）
- [ ] `docker compose up` 起動時またはデプロイスクリプト内で未適用マイグレーションを自動実行する設定を行う
- [ ] マイグレーション適用状況を確認するコマンド（`npm run migrate:status`）を実装する
- [ ] ロールバック用コマンド（`npm run migrate:rollback`）を実装する

## 技術メモ

- マイグレーションはべき等（冪等）に設計し、二重実行でエラーが起きないようにする
- マイグレーション実行ログを DB に記録し、適用済みバージョンを追跡する
- 本番環境では自動マイグレーションを無効にし、手動実行を推奨する（環境変数 `AUTO_MIGRATE=false`）
