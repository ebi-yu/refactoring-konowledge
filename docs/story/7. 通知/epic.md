# [Epic] 通知

## 概要

記事へのコメントをサイト内通知で届け、管理者からのお知らせはヘッダーで全員に強制表示する機能群。
Ph1 はアプリ内通知とシステムお知らせに絞る。Webhook 連携は Ph2 で提供する。

## Epic ストーリー

**投稿者** として、**自分の記事にコメントが付いたときにすぐ気づきたい**。なぜなら **反応に気づかないまま放置してしまうと、せっかくのやりとりが途切れてもったいないから**。

## スコープ（Story 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Must` | 1 | 投稿者として、自分の記事へのコメントをサイト内の通知で受け取りたい。なぜなら誰かが反応してくれたことにすぐ気づいて返答したいから。 | - |
| `Must` | 2 | 利用者として、管理者からのシステムお知らせをサイトヘッダーで強制表示されるのを確認したい。なぜなら重要な情報は見落とさずに全員に届けてほしいから。 | - |
| `Could` | 3 | 管理者として、記事投稿・コメントなどのイベントを Slack 等の外部サービスに Webhook で通知したい。なぜなら普段使っているチャットで知識共有の動きをキャッチしたいから。（Ph2） | - |

## スコープ外

- **メール通知**（7-2）: Ph1 除外。メール設定が必要なため運用コストが高い
- **デスクトップ通知（Push）**（7-3）: Ph1 除外。ブラウザ権限の複雑さに対して需要が不明確
- **Webhook 連携**（7-4）: Ph2 で追加。Slack 等への通知は運用が安定してから

## 受け入れ基準（Epic 完了の定義）

- [ ] 配下のすべての `Must` Story が Done になっている
- [ ] アプリ内通知・システムお知らせの E2E シナリオが通過している
- [ ] ドキュメント（ADR / API仕様）が更新されている
