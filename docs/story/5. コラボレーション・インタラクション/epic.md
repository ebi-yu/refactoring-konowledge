# [Epic] コラボレーション・インタラクション

## 概要

閲覧者が記事に対してコメント・Like・メンション・ストックなどで反応し、投稿者との双方向のやりとりを生む機能群。
チームの知識共有をより活発にするためのインタラクションを Ph1 で揃える。

## Epic ストーリー

**閲覧者** として、**記事を読んで感想を伝えたり、気になった記事をあとで読み返せるように保存したい**。なぜなら **一方的に読むだけでなく投稿者とやりとりできると知識の質が上がるし、良い記事をストックしておくと自分の仕事でも役立てられるから**。

## スコープ（Story 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Must` | 1 | 閲覧者として、記事にコメントを投稿・編集・削除したい。なぜなら疑問点を質問したり補足情報を追加してチームの理解を深めたいから。 | - |
| `Must` | 2 | 閲覧者として、良いと思った記事やコメントに Like を押したい。なぜなら言葉を使わずに「役に立った」という気持ちを投稿者に届けたいから。 | - |
| `Must` | 3 | 投稿者として、コメントや記事内で特定のユーザーを @メンションして通知したい。なぜなら見てほしい相手に確実に内容を届けたいから。 | - |
| `Must` | 4 | 閲覧者として、あとで読み返したい記事をストックリストに保存したい。なぜなら仕事で使いそうな記事をすぐに見つけられる場所にまとめておきたいから。 | - |

## スコープ外

- **アクティビティフィード**（5-5）: 実装コストに対して Ph1 では優先度が低いため除外

## 受け入れ基準（Epic 完了の定義）

- [ ] 配下のすべての `Must` Story が Done になっている
- [ ] コメント・Like・メンション・ストックの E2E シナリオが通過している
- [ ] ドキュメント（ADR / API仕様）が更新されている
