## 概要

POST/DELETE /api/stocks エンドポイントの実装。

## 親 Story

- Story: (US-3) 記事をストックリストに保存したい

## 作業内容

- [ ] `POST /api/stocks` でストック追加エンドポイントを実装する（リクエストボディに `article_id` を含める）
- [ ] `DELETE /api/stocks/:articleId` でストック削除エンドポイントを実装する
- [ ] `stocks` テーブルに `(user_id, article_id)` のユニーク制約を設定する
- [ ] 同じ記事を重複してストックしようとした場合は 409 を返す
- [ ] 記事取得 API（`GET /api/articles/:id`）のレスポンスに自分がストック済みかどうかのフラグを追加する

## 技術メモ

- ストック数は記事の人気度指標として `articles` テーブルにカウントキャッシュを持つことを検討する
- プライベートなデータのため、他のユーザーのストック一覧は公開しない
