# [Epic] 管理者機能

## 概要

管理者がユーザーを管理し、メール設定などのシステム運用に必要な機能群。
Ph1 は基本ロール（管理者/一般）のユーザー管理と条件付きメール設定に絞る。Webhook 管理は Ph2 で追加する。

## Epic ストーリー

**管理者** として、**サービスに登録されているユーザーを把握・管理し、システムが正常に動き続けるよう設定を調整したい**。なぜなら **放置するとスパムアカウントや不要なユーザーが溜まるし、メール通知が届かないとユーザーが困るから**。

## スコープ（Story 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Must` | 1 | 管理者として、登録ユーザーの一覧を確認し、作成・編集・削除ができたい。なぜなら不正なアカウントや退職者のアカウントを適切に管理してサービスを健全に保ちたいから。 | - |
| `Must` | 2 | 管理者として、ユーザーに「管理者」または「一般ユーザー」のロールを割り当てたい。なぜなら権限を分けることで誤操作や不正な変更を防ぎたいから。 | - |
| `Could` | 3 | 管理者として、Webhook エンドポイントを登録・管理したい。なぜなら通知先の外部サービスを柔軟に追加・変更したいから。（Ph2・Webhook連携と連動） | - |

## スコープ外

- **システム設定**（11-2）: Ph1 除外。サービス名等は env 変数で管理
- **LDAP 設定**（11-4）: スコープアウト。Keycloak 経由で吸収
- **テンプレート管理**（11-5）: テンプレート機能自体が除外
- **Webhook 管理**（11-6）: Ph2（Webhook 連携と連動）
- **メンテナンスモード**（11-7）: スコープアウト
- **DB 管理**（11-8）: スコープアウト。バックアップはセルフホスト Epic（13-C）で対応
