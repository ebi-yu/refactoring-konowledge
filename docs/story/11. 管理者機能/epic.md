# [Epic] 管理者機能

## 概要

管理者がユーザーを管理し、クラウドストレージ設定などのシステム運用に必要な機能群。
Ph1 はユーザー閲覧・ブロック・ロール割当とクラウドストレージ設定に絞る。Webhook 管理は Ph2 で追加する。

## Epic ストーリー

**管理者** として、**サービスに登録されているユーザーを把握・管理し、システムが正常に動き続けるよう設定を調整したい**。なぜなら **放置するとスパムアカウントや不要なユーザーが溜まるし、メール通知が届かないとユーザーが困るから**。

## スコープ（Story 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Must` | 1 | 管理者として、登録ユーザーの一覧を閲覧したい。なぜなら誰がサービスを使っているか把握して運用状況を確認したいから。 | - |
| `Must` | 2 | 管理者として、問題のあるユーザーをブロックしてサービスへのアクセスを停止したい。なぜなら不正な操作や迷惑行為をしているユーザーを即座に締め出してサービスを健全に保ちたいから。 | - |
| `Must` | 3 | 管理者として、ユーザーに「管理者」または「一般ユーザー」のロールを割り当てたい。なぜなら権限を分けることで誤操作や不正な変更を防ぎたいから。 | - |
| `Must` | 4 | 管理者として、ファイルの保存先を S3 / R2 / MinIO 等のクラウドストレージに設定したい。なぜなら DB にファイルを保存すると容量と性能の問題が出るから。 | - |
| `Could` | 5 | 管理者として、Webhook エンドポイントを登録・管理したい。なぜなら通知先の外部サービスを柔軟に追加・変更したいから。（Ph2・Webhook連携と連動） | - |

## スコープ外

- **システム設定**（11-2）: Ph1 除外。サービス名等は env 変数で管理
- **LDAP 設定**（11-4）: スコープアウト。Keycloak 経由で吸収
- **テンプレート管理**（11-5）: テンプレート機能自体が除外
- **Webhook 管理**（11-6）: Ph2（Webhook 連携と連動）
- **メンテナンスモード**（11-7）: スコープアウト
- **DB 管理**（11-8）: スコープアウト。バックアップはインフラ・運用 Epic（15）で対応

## 受け入れ基準（Epic 完了の定義）

- [ ] ユーザー管理・ロール割当の E2E シナリオが通過している
- [ ] ドキュメント（ADR / API仕様）が更新されている
