## ユーザーストーリー

| 項目 | 内容 |
|------|------|
| **Who**（誰が） | 利用者 |
| **What**（何をしたい） | GitHub / GitLab / Keycloak のアカウントでログインしたい |
| **Why**（なぜ・価値） | 新しいパスワードを作らずに使い慣れた外部アカウントで安全にサービスを利用したいから |

**ストーリー文（上記を文章化）:**

> **利用者** として、**GitHub / GitLab / Keycloak のアカウントでログインしたい**。なぜなら **新しいパスワードを作らずに使い慣れた外部アカウントで安全にサービスを利用したいから** だからだ。

## 背景・補足

複数の外部 OAuth / OIDC プロバイダー（GitHub・GitLab・Keycloak）を用いたソーシャルログインを提供することで、ユーザーは既存のアカウントをそのまま利用できる。パスワード管理の負担を減らし、セキュリティを高める。

## 受け入れ基準

- [ ] **Given** ユーザーがログインページにいる  
      **When** GitHubでサインインボタンを押す  
      **Then** OAuthフローが開始されGitHubの認証画面にリダイレクトされる

- [ ] **Given** OAuthフロー成功後  
      **When** コールバックURLが呼ばれる  
      **Then** ユーザーがDBに保存され、セッションが発行される

- [ ] **Given** ログイン済みユーザー  
      **When** プロフィールページを見る  
      **Then** GitHubから同期された表示名・アバターが表示される
