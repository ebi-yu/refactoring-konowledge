# [Epic] 認証・アカウント管理

## 概要

GitHub / GitLab / Keycloak のいずれかの OAuth / OIDC を使ってログインする機能群。
ユーザーの自己登録は行わず、外部プロバイダーのアカウントでのみサービスを利用できる。
プロフィール（表示名・アバター）はプロバイダーから自動同期し、言語・テーマはブラウザ / OS 設定に委譲する。
未ログインのゲストは公開記事を閲覧できる。

## Epic ストーリー

**投稿者** として、**使い慣れた GitHub や GitLab のアカウントでそのままログインして記事を書き始めたい**。なぜなら **新しいアカウントを作ったりパスワードを覚えるのは面倒だし、すでに使っている外部アカウントと一元管理したいから**。

## スコープ（Story 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Must` | 1 | 利用者として、GitHub / GitLab / Keycloak のアカウントでログインしたい。なぜなら新しいパスワードを作らずに使い慣れた外部アカウントで安全にサービスを利用したいから。 | - |
| `Must` | 2 | 管理者として、初回セットアップスクリプトで最初の管理者アカウントを作成したい。なぜなら誰もログインできない状態を脱して運用を開始したいから。 | - |
| `Must` | 3 | ゲストとして、ログインせずに公開記事を閲覧したい。なぜなら内容を確認してから登録するかどうか判断したいし、外部からリンクを貼っても読めるようにしたいから。 | - |
| `Could` | 4 | 開発者として、個人アクセストークンを発行して REST API を叩きたい。なぜなら CI/CD やスクリプトからトークン認証で記事を操作したいから。（Ph2・REST API と連動） | - |

## スコープ外

- **プロフィール手動編集**: 表示名・アバターは外部プロバイダーから自動同期。変更はプロバイダー側で行う
- **言語・テーマ設定**: ブラウザの言語設定・OS のダークモード設定に委譲
- **仮登録メール認証**（1-2）: 自己登録廃止のため不要
- **パスワードリセット**（1-3）: 外部プロバイダーが担うため不要
- **LDAP / AD 認証**（1-5）: Keycloak 経由で吸収
- **ログイン履歴**（1-7）: 外部プロバイダー側で管理
- **APIトークン発行**（1-8）: Ph2（REST API が Ph2 のため連動）
