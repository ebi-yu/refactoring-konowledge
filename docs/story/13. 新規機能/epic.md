# [Epic] 新規機能

## 概要

旧システムにはなく、リライトで新たに追加する機能群。
AI/RAG連携・クラウドストレージ・セルフホストサポート・Book機能・旧システム移行の5カテゴリで構成する。
クラウドストレージ・セルフホスト・Book・移行は Ph1 から提供し、AI/RAG 連携は Ph2 で追加する。

## Epic ストーリー

**システム管理者** として、**Docker Compose 一発でシステムを起動し、旧システムのデータを移行して本番環境で安定稼働させたい**。なぜなら **セットアップに何日もかかったり移行作業で情報が失われるのは困るし、チームが早く使い始められる環境を整えたいから**。

## スコープ（Story 一覧）

> 優先度: `Must` = リリース必須 / `Should` = できれば含める / `Could` = 余裕があれば

### 13-B クラウドストレージ連携（Ph1）

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Must` | 1 | 管理者として、ファイルの保存先を S3 / R2 / MinIO 等のクラウドストレージに切り替えたい。なぜなら DB に直接ファイルを保存すると容量と性能の問題が出るから。 | - |

### 13-C セルフホストサポート（Ph1）

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Must` | 2 | 管理者として、`docker compose up` だけで全サービスを起動したい。なぜなら複雑な手順なしにすぐ動く環境を手に入れたいから。 | - |
| `Must` | 3 | 管理者として、すべての設定を `.env` ファイルだけで管理したい。なぜなら設定ファイルを何箇所も編集するのは間違いのもとだから。 | - |
| `Must` | 4 | 管理者として、`/api/health` にアクセスしてサービスの死活を確認したい。なぜなら監視ツールや Kubernetes からヘルスチェックできる口が必要だから。 | - |
| `Should` | 5 | 管理者として、DB とファイルストレージのバックアップ・リストア手順とスクリプトを使いたい。なぜなら障害時に素早くデータを復元できるようにしておきたいから。 | - |
| `Should` | 6 | 管理者として、バージョンアップ時のマイグレーション手順ドキュメントを参照したい。なぜなら更新作業を安全に進めてデータを失いたくないから。 | - |

### 13-D 体系的知識（Book機能）（Ph1）

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Must` | 7 | 投稿者として、複数の記事をまとめた「本（Book）」を作成・編集・削除したい。なぜなら連載記事やチュートリアルを体系的に整理して読みやすくしたいから。 | - |
| `Must` | 8 | 投稿者として、本にチャプターを追加・並び替え・削除したい。なぜなら章立てで内容を構成して読者が順番に理解できるようにしたいから。 | - |
| `Must` | 9 | 投稿者として、本単位で公開 / 非公開 / 限定公開を設定したい。なぜなら完成した本だけを読者に届けて、書きかけのものは隠しておきたいから。 | - |

### 13-E 旧システム移行（Ph1）

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Must` | 10 | 管理者として、旧システムのユーザーデータをバッチで新システムに移行したい。なぜなら既存メンバーが再登録なしにそのまま使い続けられるようにしたいから。 | - |
| `Must` | 11 | 管理者として、旧システムの記事データ・タグを移行したい。なぜなら過去に蓄積した知識を新システムでもそのまま活用したいから。 | - |
| `Must` | 12 | 管理者として、旧システムの添付ファイル（DB の BYTEA）をクラウドストレージに移行したい。なぜなら画像や資料が失われると記事の内容が不完全になるから。 | - |
| `Must` | 13 | 管理者として、コメントと Like のデータを移行したい。なぜなら記事へのリアクションも含めて過去の知識共有の文脈を保ちたいから。 | - |
| `Must` | 14 | 管理者として、移行前後のレコード数・差分をレポートで確認したい。なぜなら移行漏れや変換エラーが起きたときにすぐ気づいて対処したいから。 | - |

### 13-D Book機能（Ph2追加分）

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Could` | 15 | 閲覧者として、本の目次をサイドバーで見ながら前後チャプターへ移動したい。なぜなら長い本を章立てで快適に読み進めたいから。（Ph2・UX最適化） | - |

### 13-E 移行（Ph2追加分）

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Could` | 16 | 管理者として、旧システムのグループ定義とロール割当を Keycloak へ移行したい。なぜなら既存の権限設定をそのまま引き継いで運用を継続したいから。（Ph2・Keycloak連携と連動） | - |

### 13-A AI / RAG 連携（Ph2）

| 優先度 | # | ストーリー（一文） | Issue |
|--------|---|-------------------|-------|
| `Could` | 17 | 管理者として、記事を外部 RAG（AWS Bedrock Knowledge Base 等）に自動同期したい。なぜなら AI が常に最新の社内知識に基づいて回答できるようにしたいから。（Ph2） | - |
| `Could` | 18 | 利用者として、社内ナレッジベースに対して AI に質問して回答を得たい。なぜなら大量の記事を検索せずに自然言語で知りたいことを聞けると時短になるから。（Ph2） | - |
