## 概要

POST /api/articles エンドポイントの実装（タイトル・本文・公開範囲の受け取りとDB保存）。

## 親 Story

- Story: (US-1) タイトルとMarkdown本文で記事を作成したい

## 作業内容

- [ ] `POST /api/articles` エンドポイントを実装する
- [ ] リクエストボディで `title`・`body`（Markdown）・`visibility` を受け取る
- [ ] バリデーション（タイトル必須・最大文字数等）を実装する
- [ ] articles テーブルに記事を保存し、作成した記事の ID を含む 201 レスポンスを返す
- [ ] 未認証リクエストには 401 を返す

## 技術メモ

- `author_id` はセッションから取得したログインユーザーの ID を使用する
- `created_at` / `updated_at` は DB のデフォルト値で自動設定する
